# دليل نقل قاعدة البيانات

هذا الدليل يشرح كيفية نقل محتوى قاعدة البيانات من Supabase إلى قاعدة بيانات أخرى.

## المتطلبات

- Node.js مثبت على جهازك
- بيانات الوصول لقاعدة البيانات المصدر (الحالية)
- بيانات الوصول لقاعدة البيانات الهدف (الجديدة)

## خطوات النقل

### 1. تحضير قاعدة البيانات الهدف

تأكد من أن قاعدة البيانات الهدف تحتوي على نفس الهيكل (الجداول والأعمدة):

```sql
-- الجداول المطلوبة:
- categories
- subcategories
- services
- product_images
- product_sizes
- banners
- store_settings
- testimonials
```

يمكنك تشغيل نفس الهجرات من مجلد `supabase/migrations/` على قاعدة البيانات الهدف.

### 2. إعداد متغيرات البيئة

أنشئ ملف `.env.migration` في جذر المشروع:

```env
SOURCE_SUPABASE_URL=https://your-source-project.supabase.co
SOURCE_SUPABASE_KEY=your-source-anon-key
TARGET_SUPABASE_URL=https://your-target-project.supabase.co
TARGET_SUPABASE_KEY=your-target-anon-key
```

**ملاحظة أمان مهمة:**
- استخدم مفاتيح الخدمة (Service Role Keys) للحصول على صلاحيات كاملة
- لا تشارك هذه المفاتيح مع أحد
- احذف الملف بعد انتهاء النقل

### 3. تشغيل سكريبت النقل

```bash
# تثبيت المتطلبات إذا لم تكن مثبتة
npm install

# تشغيل السكريبت
npx ts-node scripts/migrate-database.ts
```

أو إذا كنت تستخدم Node.js مباشرة:

```bash
node --loader ts-node/esm scripts/migrate-database.ts
```

### 4. التحقق من البيانات

بعد انتهاء النقل، تحقق من:

1. عدد السجلات في كل جدول
2. سلامة البيانات والعلاقات بين الجداول
3. الصور والملفات المرفقة

## الجداول المنقولة

| الجدول | الوصف |
|--------|-------|
| `categories` | فئات المنتجات/الخدمات |
| `subcategories` | الفئات الفرعية |
| `services` | الخدمات والمنتجات |
| `product_images` | صور المنتجات |
| `product_sizes` | أحجام المنتجات والأسعار |
| `banners` | البانرات والإعلانات |
| `store_settings` | إعدادات المتجر |
| `testimonials` | آراء العملاء |

## استكشاف الأخطاء

### خطأ: "Missing environment variables"
- تأكد من وجود ملف `.env.migration`
- تحقق من أن جميع المتغيرات موجودة وصحيحة

### خطأ: "Permission denied"
- استخدم مفاتيح الخدمة (Service Role Keys) بدلاً من مفاتيح Anon
- تأكد من أن المفاتيح صحيحة

### خطأ: "Table does not exist"
- تأكد من تشغيل الهجرات على قاعدة البيانات الهدف أولاً
- تحقق من أن الجداول موجودة في قاعدة البيانات الهدف

## خيارات متقدمة

### نقل جدول واحد فقط

عدّل السكريبت وغيّر قائمة الجداول:

```typescript
const tables = ['services']; // نقل جدول واحد فقط
```

### عدم حذف البيانات الموجودة

علّق السطر التالي في السكريبت:

```typescript
// await targetClient.from(table).delete().neq('id', '');
```

### نقل البيانات مع الحفاظ على الهويات

إذا كنت تريد الحفاظ على نفس الهويات (IDs)، تأكد من:
1. أن قاعدة البيانات الهدف فارغة
2. أن الأعمدة من نوع UUID أو Integer بنفس الطريقة

## الدعم

إذا واجهت مشاكل:
1. تحقق من السجلات (logs) في الكونسول
2. تأكد من اتصالك بالإنترنت
3. تحقق من صلاحيات المفاتيح المستخدمة
