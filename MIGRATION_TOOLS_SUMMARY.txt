================================================================================

ثم أدخل بيانات قاعدتي البيانات وسيتم النقل تلقائياً!

================================================================================
                                  خطوات النقل
================================================================================

1. التحضير
   - اقرأ SETUP_TARGET_DATABASE.md
   - أنشئ مشروع Supabase جديد
   - طبّق جميع الهجرات

2. النقل
   - شغّل: npm run migrate:interactive
   - أو استخدم: npm run migrate

3. التحقق
   - اتبع VERIFY_MIGRATION.md
   - تحقق من جميع البيانات

4. التبديل
   - اتبع SWITCH_DATABASE.md
   - حدّث ملف .env

5. النشر
   - اختبر محلياً
   - نشّر التحديثات

✓ أضفه إلى .gitignore إذا لم يكن موجوداً
✓ احتفظ بنسخة احتياطية من البيانات

================================================================================
                                  الأسئلة الشائعة
================================================================================

س: من أين أبدأ؟
ج: اقرأ START_HERE.md ثم شغّل npm run migrate:interactive

س: كم من الوقت يستغرق النقل؟
ج: عادة من ثوانٍ إلى دقائق حسب حجم البيانات

س: هل يمكن الرجوع للخلف؟
ج: نعم، اقرأ ROLLBACK_MIGRATION.md

س: ماذا لو حدث خطأ؟
ج: اقرأ الدليل المناسب لاستكشاف الأخطاء

س: هل البيانات آمنة؟
ج: نعم، استخدم Service Role Keys وحافظ على سرية المفاتيح
