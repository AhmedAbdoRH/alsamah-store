# الرجوع للخلف - Rollback

إذا حدثت مشكلة أثناء النقل أو بعده، اتبع هذه الخطوات للرجوع للخلف.

## 1. الرجوع السريع

### إذا لم تكن قد حدّثت الموقع بعد

```bash
# لا تفعل شيء، ما زال الموقع يستخدم قاعدة البيانات القديمة
# يمكنك إعادة محاولة النقل
npm run migrate:interactive
```

### إذا حدّثت الموقع بالفعل

```bash
# استعد ملف .env القديم
git checkout .env

# أعد تشغيل الموقع
npm run dev
```

## 2. الرجوع من قاعدة البيانات الجديدة

### الخطوة 1: تحديث ملف `.env`

```env
# استخدم بيانات قاعدة البيانات القديمة
VITE_SUPABASE_URL="https://your-old-project.supabase.co"
VITE_SUPABASE_ANON_KEY="your-old-anon-key"
VITE_SUPABASE_SERVICE_ROLE_KEY="your-old-service-role-key"
```

### الخطوة 2: أعد تشغيل الموقع

```bash
npm run dev
```

### الخطوة 3: تحقق من أن الموقع يعمل

- [ ] الموقع يحمّل بدون أخطاء
- [ ] البيانات تظهر بشكل صحيح
- [ ] الوظائف تعمل بشكل طبيعي

## 3. حذف قاعدة البيانات الجديدة (اختياري)

إذا كنت متأكداً من أنك لن تحتاج قاعدة البيانات الجديدة:

### عبر Supabase Dashboard

1. اذهب إلى [Supabase Dashboard](https://app.supabase.com)
2. اختر المشروع الجديد
3. اذهب إلى **Settings** → **General**
4. انقر **Delete Project**
5. أدخل اسم المشروع للتأكيد
6. انقر **Delete**

### عبر Supabase CLI

```bash
supabase projects delete --project-ref new-project-ref
```

## 4. استعادة البيانات المحذوفة

إذا حذفت البيانات من قاعدة البيانات الجديدة عن طريق الخطأ:

### من النسخة الاحتياطية

```bash
# إذا كان لديك نسخة احتياطية
supabase db restore --project-ref new-project-ref --backup-id backup-id
```

### من قاعدة البيانات القديمة

```bash
# أعد تشغيل النقل
npm run migrate:interactive
```

## 5. استكشاف الأخطاء

### الموقع لا يعمل بعد الرجوع

1. تحقق من ملف `.env`:
```bash
cat .env
```

2. تأكد من أن المفاتيح صحيحة

3. افتح Developer Console (F12) وتحقق من الأخطاء

4. أعد تشغيل خادم التطوير:
```bash
npm run dev
```

### البيانات مفقودة

1. تحقق من أن قاعدة البيانات القديمة لا تزال موجودة
2. تحقق من أن البيانات لم تُحذف من قاعدة البيانات القديمة
3. استعد من النسخة الاحتياطية إذا أمكن

### الملفات مفقودة

1. تحقق من أن الملفات لا تزال في Storage القديم
2. استعد من النسخة الاحتياطية للملفات
3. أعد تحميل الملفات يدوياً

## 6. منع المشاكل في المستقبل

### 1. اختبر قبل النقل

```bash
# تحقق من أن قاعدة البيانات الجديدة جاهزة
# تحقق من أن جميع الهجرات تم تطبيقها
```

### 2. احتفظ بنسخة احتياطية

```bash
# قبل النقل، احفظ نسخة احتياطية من قاعدة البيانات القديمة
supabase db backup --project-ref old-project-ref
```

### 3. اختبر على بيئة تطوير أولاً

```bash
# استخدم بيئة تطوير منفصلة للاختبار
# تأكد من أن كل شيء يعمل قبل الانتقال للإنتاج
```

### 4. وثّق كل شيء

```bash
# احفظ:
# - بيانات الاتصال القديمة
# - بيانات الاتصال الجديدة
# - تاريخ النقل
# - أي مشاكل واجهتها
```
