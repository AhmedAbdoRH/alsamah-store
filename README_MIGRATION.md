# 🚀 نقل قاعدة البيانات - Database Migration

مجموعة شاملة من الأدوات والأدلة لنقل محتوى قاعدة البيانات من Supabase إلى قاعدة بيانات أخرى.

---

## ⚡ البدء السريع (3 خطوات)

### 1️⃣ شغّل الأداة التفاعلية
```bash
npm run migrate:interactive
```

### 2️⃣ أدخل بيانات قاعدتي البيانات
- رابط Supabase المصدر
- مفتاح الخدمة للمصدر
- رابط Supabase الهدف
- مفتاح الخدمة للهدف

### 3️⃣ انتظر انتهاء النقل
```
✅ تم استخراج البيانات
✅ تم إدراج البيانات
✨ اكتمل النقل بنجاح!
```

## 🛠️ الأدوات المتاحة

### سكريبتات النقل

```bash
# الطريقة التفاعلية (الموصى به)
npm run migrate:interactive

# باستخدام ملف البيئة
npm run migrate
```

### ملفات السكريبتات

- `scripts/migrate-database.js` - سكريبت النقل الأساسي
- `scripts/interactive-migrate.js` - أداة تفاعلية سهلة
- `scripts/migrate-database.ts` - نسخة TypeScript

### ملفات التكوين

- `.env.migration.example` - مثال على ملف المتغيرات

## 🎯 خطوات النقل الموصى بها

### المرحلة 1: التحضير (30 دقيقة)
1. اقرأ [START_HERE.md](./START_HERE.md)
2. اقرأ [GET_CREDENTIALS.md](./GET_CREDENTIALS.md)
3. اقرأ [SETUP_TARGET_DATABASE.md](./SETUP_TARGET_DATABASE.md)

### المرحلة 2: النقل (5-15 دقيقة)
```bash
npm run migrate:interactive
```

### المرحلة 3: التحقق (20 دقيقة)
1. اقرأ [VERIFY_MIGRATION.md](./VERIFY_MIGRATION.md)
2. تحقق من جميع البيانات

### المرحلة 4: التبديل (10 دقائق)
1. اقرأ [SWITCH_DATABASE.md](./SWITCH_DATABASE.md)
2. حدّث ملف `.env`
3. أعد تشغيل الخادم

### المرحلة 5: النشر (15 دقيقة)
1. اختبر الموقع محلياً
2. نشّر التحديثات

**الوقت الإجمالي: 1.5-2 ساعة**

## 🔐 نصائح الأمان

✅ استخدم **Service Role Keys** فقط (لديها صلاحيات كاملة)
✅ **لا تشارك** ملف `.env.migration` مع أحد
✅ **احذف** الملف بعد انتهاء النقل
✅ **أضفه إلى `.gitignore`** إذا لم يكن موجوداً
✅ احتفظ بنسخة احتياطية من البيانات

## ❓ الأسئلة الشائعة

**س: من أين أبدأ؟**
ج: اقرأ [START_HERE.md](./START_HERE.md) ثم شغّل `npm run migrate:interactive`

**س: كيف أحصل على المفاتيح؟**
ج: اقرأ [GET_CREDENTIALS.md](./GET_CREDENTIALS.md)

**س: كم من الوقت يستغرق النقل؟**
ج: عادة من 5-15 دقيقة حسب حجم البيانات

**س: هل يمكن نقل جدول واحد فقط؟**
ج: نعم، عدّل قائمة الجداول في السكريبت

**س: هل يمكن الرجوع للخلف؟**
ج: نعم، اقرأ [ROLLBACK_MIGRATION.md](./ROLLBACK_MIGRATION.md)

**س: ماذا لو حدث خطأ؟**
ج: اقرأ الدليل المناسب لاستكشاف الأخطاء

**س: هل البيانات آمنة؟**
ج: نعم، استخدم Service Role Keys وحافظ على سرية المفاتيح
