# ملخص نقل قاعدة البيانات

## 📋 الملفات المُنشأة

تم إنشاء الملفات والأدوات التالية لمساعدتك في نقل البيانات:

### 1. أدوات النقل
- **`scripts/migrate-database.js`** - سكريبت النقل الأساسي
- **`scripts/interactive-migrate.js`** - أداة تفاعلية سهلة الاستخدام
- **`scripts/migrate-database.ts`** - نسخة TypeScript (اختياري)

### 2. ملفات التكوين
- **`.env.migration.example`** - مثال على ملف المتغيرات

### 3. أدلة التعليمات
- **`QUICK_MIGRATION.md`** - دليل البدء السريع ⭐
- **`DATABASE_MIGRATION_GUIDE.md`** - دليل شامل
- **`SETUP_TARGET_DATABASE.md`** - إعداد قاعدة البيانات الهدف
- **`SWITCH_DATABASE.md`** - تبديل الموقع للاتصال بقاعدة جديدة

### 4. تحديثات package.json
تم إضافة أوامر جديدة:
```json
"migrate": "node scripts/migrate-database.js",
"migrate:interactive": "node scripts/interactive-migrate.js"
```

## 🚀 خطوات النقل السريعة

### الخطوة 1: إعداد قاعدة البيانات الهدف
```bash
# اقرأ SETUP_TARGET_DATABASE.md
# تأكد من تطبيق جميع الهجرات
```

### الخطوة 2: نقل البيانات (اختر واحدة)

**الطريقة الأسهل - التفاعلية:**
```bash
npm run migrate:interactive
```

**أو باستخدام ملف البيئة:**
```bash
cp .env.migration.example .env.migration
# عدّل .env.migration بإضافة بيانات قاعدتي البيانات
npm run migrate
```

### الخطوة 3: تبديل الموقع للاتصال بقاعدة جديدة
```bash
# اقرأ SWITCH_DATABASE.md
# حدّث ملف .env بالمفاتيح الجديدة
npm run dev
```

## 🔐 نصائح الأمان

- ✅ استخدم **Service Role Keys** فقط (لديها صلاحيات كاملة)
- ✅ **لا تشارك** ملف `.env.migration` مع أحد
- ✅ **احذف** ملف `.env.migration` بعد انتهاء النقل
- ✅ **أضفه إلى `.gitignore`** إذا لم يكن موجوداً

## ❓ الأسئلة الشائعة

### س: هل ستُحذف البيانات القديمة؟
ج: نعم، السكريبت يحذف البيانات الموجودة في قاعدة البيانات الهدف قبل الإدراج.

### س: هل يمكن نقل جدول واحد فقط؟
ج: نعم، عدّل قائمة الجداول في السكريبت.

### س: ماذا لو حدث خطأ أثناء النقل؟
ج: السكريبت يتابع مع الجداول الأخرى ويعرض رسائل خطأ واضحة.

### س: كم من الوقت يستغرق النقل؟
ج: يعتمد على حجم البيانات، عادة من ثوانٍ إلى دقائق.

## 📞 الدعم والمساعدة
